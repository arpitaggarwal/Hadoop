Hadoop Cluster

What is Hadoop Cluster?

Hadoop cluster is a group of computers working together. Usually runs "HDFS" and "MapReduce". A node is an individual computer in the cluster.Master nodes manage distribution of work and data to slave nodes. A daemon is a program running on a node.

Hadoop Daemons

It's grouped into 2 : HDFS and MapReduce.

HDFS daemons

  1. NameNode – holds the metadata for HDFS. Typically two on a production cluster: one active, one standby.
  2. DataNode – holds the actual HDFS data. One per slave node.

MapReduce daemons

It's grouped into 2 : MapReduce 1 and MapReduce 2.

MapReduce 1 (MRv1 or "Classic MapReduce")

  • It uses a "JobTracker/TaskTracker" architecture.
  • One "JobTracker" per cluster and also limits cluster size to about 4000 nodes.
  • It uses Slots on slave nodes designated for Map or Reduce tasks.

MapReduce 2 (MRv2)

  • It uses a "ResourceManager/NodeManager" architecture.
  • Built on top of "YARN"(Yet"Another"Resource"Negotiator) which increases scalability of cluster. Cluster nodes or
    resources can be used for any type of task, which further improves cluster utilization.
  • Provided support for non-MapReduce jobs.
